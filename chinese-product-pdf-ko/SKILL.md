# 중국어 제품 PDF 한국어화

## 목표

중국어 제품 설명서 또는 사양서 PDF를 한국어 사용자가 바로 검토할 수 있는 A4 세로 PDF로 재작성함. 원문 수치, 단위, 모델명, URL, 날짜는 유지하고 설명 문장과 항목명만 한국어로 옮김.

## 작업 흐름

1. 원본 파일 확인
   - 대상 PDF를 `fitz`로 열어 페이지 수, 텍스트 추출 가능 여부, 이미지 포함 여부를 확인함.
   - 텍스트형 PDF이면 `scripts/extract_pdf_text.py`로 페이지별 텍스트를 추출함.
   - 스캔형 PDF이면 OCR 가능 도구를 먼저 확인하고, OCR 환경이 없으면 사용자에게 원본 이미지 인식 한계를 알림.

2. 번역 구조화
   - 제품 기본 정보, 사양 파라미터, 포장 구성, 판매 및 보증, 원문 표기 정보처럼 원문 구조에 맞는 섹션만 구성함.
   - 원문에 없는 구매 제안, 의사결정 요청, 출처 유보 섹션은 추가하지 않음.
   - 브랜드명과 모델명은 원문 표기를 우선 사용하고, 중국어 명사가 제품군 설명에 해당하면 한국어 병기를 적용함.
   - URL은 원문 그대로 유지함.

3. PDF 생성
   - 번역 내용을 JSON으로 정리한 뒤 `scripts/make_korean_pdf_from_json.py`를 실행함.
   - 기본 출력은 A4 세로, 검정 글씨, 맑은 고딕 계열 서식으로 구성함.
   - 이미지 삽입이 필요하면 원본 종횡비를 보존하고 한 축만 지정해 배치함.

4. 검증
   - 생성 PDF를 다시 `fitz`로 열어 페이지 수와 주요 키워드 포함 여부를 확인함.
   - 모델명, 수치, 단위, 보증 기간, 생성일, URL이 원문과 일치하는지 비교함.
   - 중간 산출 PDF가 있으면 최종본만 남기고 임시 파일을 정리함.

## JSON 입력 형식

`make_korean_pdf_from_json.py`는 아래 구조를 입력으로 받음.

```json
{
  "title": "제품 설명서 한국어 번역본",
  "output": "제품설명서_한국어.pdf",
  "sections": [
    {
      "heading": "제품 기본 정보",
      "rows": [
        ["브랜드", "원문 브랜드명"],
        ["모델", "원문 모델명"]
      ]
    },
    {
      "note": "원문에 포함된 주의 사항을 한국어로 옮김."
    }
  ]
}
```

## 문체와 표기

- 본문 문장은 가능한 한 `~음` 체로 맞춤.
- 제품명, 모델명, 수치, 단위, 전압, 포장, 면적, URL은 임의 환산하거나 요약하지 않음.
- 치수 표기의 `*`가 원문에서 곱하기 의미이면 `x`로 바꿀 수 있으나 값 자체는 바꾸지 않음.
- 자동 번역기가 없으면 모델이 직접 번역하되, 불확실한 기술 용어는 원문과 대조해 보수적으로 옮김.

## 포함 스크립트

- `scripts/extract_pdf_text.py`: PDF의 페이지별 텍스트와 기본 메타 정보를 추출함.
- `scripts/make_korean_pdf_from_json.py`: 구조화한 한국어 JSON을 정식 PDF로 생성함.
