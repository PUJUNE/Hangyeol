# code-narrator-ko — 코드 한국어 해설가

## 목적

코드 폴더를 받아 비개발자가 **한 줄도 빼놓지 않고 사이트가 굴러가는 원리**를 머리에 잡을 수 있도록 한국어 해설본을 자동 생성. 사용자가 AI에게 "이 부분을 이렇게 바꿔줘"라고 정확히 의지를 표현할 수 있게 만드는 게 최종 목표.

## 호출 방법

- `/code-narrator-ko` — 현재 작업 폴더(cwd)를 자동 대상.
- `/code-narrator-ko <폴더경로>` — 명시 지정.
- 자연어 트리거 — "비개발자용으로 풀어줘", "이 폴더 코드 한국어로 해설해줘", "코드 풀이 만들어줘".

## 작업 흐름 (사용자 개입 0회)

다음 단계를 순서대로 자동 실행. 중간에 사용자에게 질문하지 않음. 단, **시작 직전에 발견된 파일 분류 결과를 한 번 보고**(이의 없으면 자동 진행).

### 0. 사전 점검

- 입력 폴더가 존재하고 코드 파일이 있는지 확인. 없으면 사용자에게 폴더 경로를 다시 물어봄.
- 입력 폴더에 이미 `주석본/`이 있으면 사용자에게 안내 — 덮어쓰기·이어쓰기·취소 중 선택. (이 한 번만 예외적 사용자 개입)

### 1. 파일 스캔 + 자동 분류

- `Glob`/`Bash find` 등으로 입력 폴더의 모든 파일 수집.
- 각 파일을 **정상 처리 / 요약만 / 건너뜀** 셋 중 하나로 자동 분류 — 규칙은 `references/file-type-handlers.md` 참조.
- 분류 결과를 표 한 번 짧게 보고. 사용자 응답 안 기다리고 진행.

### 2. 출력 폴더 생성

```
<입력폴더>/주석본/
├── 00_아키텍처.md          (마지막에 작성)
├── 01_읽는법.md            (1단계 후 작성)
├── 02_파일별_요약/         (원본 폴더 구조 그대로 미러링)
└── 03_라인주석/            (원본 폴더 구조 그대로 미러링)
```

원본 파일은 절대 수정하지 않음. 모든 산출물은 `주석본/`에만 생성.

### 3. `01_읽는법.md` 작성

- `references/readme-seed.md`의 기본 골격을 가져와 **실제로 발견된 파일 타입에만 맞춰 축소·확장**.
- 발견된 모든 파일 타입의 사전 포함 (HTML·CSS·JS·JSON·XML·MD·TXT 등).
- 자주 나오는 기호 사전, 파일별 요약·라인 주석본 읽는 법, 권장 학습 순서.

### 4. 각 파일 처리 (TaskList로 진행 추적)

처리 순서: **작은 데이터·메타 → 스타일 → 자바스크립트 → 작은 페이지 → 큰 페이지 → 디자인 시스템·참조 자료**. 작은 것부터 처리해 사용자가 빠르게 진척을 확인할 수 있게.

각 파일에 대해:
- **정상 처리**: 라인 주석본(`03_라인주석/<원본경로>`) + 파일별 요약(`02_파일별_요약/<원본경로>.md`) 둘 다 작성.
- **요약만**: 파일별 요약만 작성. 머리에 "라인 주석 생략 — 이유"를 명시.
- **건너뜀**: 작성 안 함. 마지막 보고서에만 기록.

라인 주석 양식, 파일별 요약 양식은 `references/output-templates.md` + `references/conventions.md` 참조. 비유는 `references/analogy-pool.md`의 메타포 풀에서 일관된 것을 사용.

### 5. `00_아키텍처.md` 작성 (마지막)

모든 파일 처리가 끝난 후 통합 지도 작성. 12개 절 구조는 `references/output-templates.md`의 아키텍처 템플릿 참조. 핵심:
- 4개 층(데이터 → 디자인 토큰 → 공통 골격 → 페이지) 다이어그램
- 페이지 지도 (사이트맵 기반)
- 한 페이지가 화면에 뜨는 N단계
- 데이터 → 페이지 흐름 (구체 예시 1개)
- 4가지 변경의 진입점 표 (콘텐츠/디자인/동작/구조)
- 자주 일어나는 변경의 진입점 한눈 보기 표
- 비개발자 권장 학습 순서

### 6. 완료 보고

산출물 위치 + 처리 통계(라인 주석 N개, 요약 N개, 건너뛴 파일 N개) + 권장 학습 순서를 한 번에 사용자에게 안내.

## 컨벤션 요약 (상세는 references/conventions.md)

- **라인 주석**: 줄당 1줄, 비유 없이 용어 풀이만.
- **파일별 요약**: ① 전체 요약(비유 적극) ② 의미 단위 묶음 ③ 수정 질문 표 ④ 함께 보면 좋은 파일.
- **어미**: 모든 산출물 `~음` 체 (CLAUDE.md의 docx 규칙을 md에도 일관 적용).
- **파일 타입별 주석 기호**:
  - HTML/XML/MD: `<!-- 주석 -->`
  - CSS: `/* 주석 */`
  - JS/JSON: `// 주석` (JSON 라인 주석본은 사람용 산출물이라 표준 JSON 파서가 못 읽어도 무방)
  - TXT/robots.txt: `# 주석`

## references/ 안내

이 스킬은 다음 4개 참조 파일에 의존:

- `references/conventions.md` — 산출물 양식 규칙 전체.
- `references/file-type-handlers.md` — 파일 분류 자동 판정 규칙.
- `references/output-templates.md` — 라인 주석본·파일 요약·아키텍처 문서 템플릿.
- `references/analogy-pool.md` — 일관된 비유 메타포 풀 (호텔·책·공장·자료실 등).
- `references/readme-seed.md` — `01_읽는법.md`의 기본 골격.

## 향후 확장 (Q2의 단계적 확장)

- **R&D 코드 지원**: `references/file-type-handlers.md`에 `.py`, `.ipynb`, `.m`(MATLAB), `.r` 규칙만 추가하면 자동 활성화. 컨벤션·템플릿·비유 풀은 그대로 재사용.
- **새 파일 타입의 비유**: `analogy-pool.md`에 추가 메타포만 정의.
- **다른 언어**: SKILL.md description의 트리거를 영어로 추가하고 conventions의 어미 규칙을 분기.

## 주의 사항

- 원본 파일은 어떤 경우에도 수정·삭제 안 함.
- `주석본/` 외 다른 폴더에 산출물 생성 금지.
- 한 파일 처리 중 오류 발생 시 그 파일은 건너뛰고 보고에 기록, 다른 파일 진행은 멈추지 않음.
- 입력 파일 1000줄 이상 + 외부 의존 5개 이상이면 자동으로 "요약만"으로 강등 (학습 가치 < 처리 비용).
- 비유는 카테고리별로 일관 사용. 같은 파일 안에서 호텔 비유와 공장 비유를 섞지 않음.
