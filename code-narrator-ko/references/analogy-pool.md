# analogy-pool.md — 일관된 비유 메타포 풀

이 문서는 파일별 요약에서 사용할 **일상 비유 메타포**를 카테고리별로 정리. 한 파일 요약 안에서 여러 카테고리를 섞지 말고, 파일 타입에 맞는 카테고리 하나를 선택해 일관 적용.

---

## 1. 핵심 카테고리

### A. 호텔 (웹사이트의 메인 메타포)

가장 큰 시스템·페이지 단위 설명에 권장.

| 코드 개념 | 호텔 비유 |
|---|---|
| 웹사이트 | 호텔 한 채 |
| 페이지 | 객실 / 층 / 구역 |
| 헤더 | 로비 천장의 명패 / 입구 위 안내띠 |
| 푸터 | 호텔 안내데스크의 회사 정보 / 출구의 마지막 인사 |
| 메뉴 | 객실 안내 표지판 |
| 사이드바 | 객실 입구의 호수판 |
| 빵부스러기(breadcrumb) | 호텔 내 위치 표시 (1F > A동 > 101호) |
| 로그인/사용자 영역 | 안내데스크 / 체크인 카운터 |
| 폼 | 신청서 카운터 / 예약 양식 |
| 검색 | 인덱스 단말기 / 안내 도우미 |

| 로딩 | 객실 준비 중 안내 |

| 파일 타입 | 1순위 비유 | 2순위 (보조) |

| HTML 페이지 | A 호텔 (페이지=객실) | B 책 (문서 구조) |
| HTML 디자인 시스템 | E 인테리어 매뉴얼 | A 호텔 (시각 정체성) |
| CSS 토큰/공통 | E 인테리어 매뉴얼 | — |
| CSS 페이지 전용 | A 호텔 (객실 인테리어) | E 인테리어 |
| JavaScript 동작 | C 공장 / 작업대 | F 우편 (요청/응답) |
| JavaScript 자동 주입 | A 호텔 (자동 명패 공장) | C 공장 |
| JSON 데이터 | D 자료실 / 카드 박스 | — |
| JSON 사전(i18n) | D 자료실 (사전·번역집) | — |
| Markdown README | B 책 (안내문) | — |

## 3. 비유 작성 규칙

### 3-1. 시작은 비유, 끝은 구체

```markdown
이 파일은 **사이트의 첫 인상이자 회사 한 권 압축본**임. 비유하면 **호텔 1층 로비
+ 회사 소개 영상관 + 카탈로그 진열대 + 안내 데스크가 한 공간에 정렬돼 있는 메인 홀**임.
1119줄 안에 7개 섹션이 위에서 아래로 줄지어 있고, 각 섹션은
좌측 사이드바(키커·H2·리드) + 우측 메인(카드·통계·타임라인)의 일관된 L자형 레이아웃을 따름.
```

비유 → 구체 수치(1119줄, 7개 섹션) → 구조 설명. 좋은 패턴.

### 3-2. 한 단락에 한 비유만

✗ 나쁨: "호텔의 로비이자 책의 표지이며 공장의 출입문"
✓ 좋음: "호텔의 로비. 손님(사용자)이 첫 인상을 받는 곳."

### 3-3. 비유는 의미 단위마다 바꿔도 됨

같은 파일 안의 ① 묶음에 호텔 비유, ② 묶음에 자료실 비유는 가능. 단, 각 묶음 안에서는 일관 사용.

### 3-4. 너무 멀리 가지 않기

✗ 나쁨: "이 함수는 호텔에서 손님이 객실 키를 받아 엘리베이터를 타고 올라가서 객실 문을 열고 들어가 침대에 눕는 과정의 마지막 한 발 디딤"
✓ 좋음: "이 함수는 객실 문을 여는 한 단계."

### 3-5. 비유 자체가 설명을 더 길게 만들면 안 씀

비유의 목적은 **이해를 빠르게 하는 것**. 비유 설명에 줄이 더 들어가면 비유를 뺌.

## 5. 비유 선택 의사결정 트리

```
파일을 받았을 때:
  ├─ HTML이면?
  │   ├─ 페이지(/, /products 등)? → A 호텔 (객실)
  │   ├─ 슬라이드/디자인 시스템? → B 책 + E 인테리어
  │   └─ 폼이 핵심? → A 호텔 (카운터)
  │
  ├─ CSS이면?
  │   ├─ 토큰/공통? → E 인테리어 매뉴얼
  │   └─ 페이지 전용? → E + A 혼합
  │
  ├─ JS이면?
  │   ├─ 동작/이벤트 위주? → C 공장 / 작업대
  │   ├─ 데이터 페칭? → F 우편
  │   └─ 자동 주입? → A + C
  │
  ├─ JSON이면? → D 자료실 / 카드 박스
  │
  ├─ MD이면? → B 책
  │
  └─ XML/TXT? → 컨텍스트에 따라 (sitemap → B 또는 A; robots → A)
```
