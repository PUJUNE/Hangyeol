# file-type-handlers.md — 파일 분류 자동 판정 규칙

이 문서는 입력 폴더의 파일을 **정상 처리 / 요약만 / 건너뜀** 셋 중 하나로 자동 분류하는 규칙임. 사용자 개입 없이 적용.

---

## 1. 분류 결과 정의

| 분류 | 의미 | 산출물 |
|---|---|---|
| **정상 처리** | 사용자가 직접 작성한 코드 | 라인 주석본 + 파일별 요약 |
| **요약만** | 자동 생성·외부 백업·소스 모형 | 파일별 요약만 (라인 주석 생략 이유 명시) |
| **건너뜀** | 바이너리·이미지·아카이브 | 산출물 없음, 보고서에만 기록 |

---

## 3. 요약만 — 자동/외부 코드

다음 조건 중 **하나라도** 해당되면 "요약만"으로 분류.

### 3-1. 헤더 마커

- 첫 5줄 안에 다음 중 하나 발견:
  - `<!-- saved from url=` (브라우저 "다른 이름으로 저장" 흔적)
  - `<!-- Generated by` 또는 `# Generated by` 또는 `// Generated by`
  - `<!-- DO NOT EDIT` 또는 `// AUTO-GENERATED`
  - `<!doctype html>` 다음 줄에 `class="js cssanimations"` 같은 CMS 자동 클래스

### 3-2. 폴더 위치

- 경로 어디에든 다음 폴더명 포함:
  - `uploads/`, `_uploads/`, `imports/`
  - `backup/`, `archive/`, `_archive/`
  - `vendor/`, `vendors/`, `node_modules/`
  - `dist/`, `build/`, `_compiled/` (단, 이 안의 파일만 단순 요약. 그 자체가 빌드 산출물이라 학습 가치 낮음)
  - `third_party/`, `external/`

### 3-3. 크기·의존도

- 파일이 **1000줄 이상** + **외부 라이브러리·CDN 호출 5개 이상** (예: jQuery, Tailwind, Bootstrap, Font Awesome 등이 동시에 호출)
- 파일이 **5000줄 이상** (무조건 요약만, 학습 가치 < 처리 비용)

### 3-4. 운영 사이트의 원본 모형

- 같은 콘텐츠의 **production 버전이 별도 존재**하는 경우 (디자인 시스템 폴더에 있는 mockup 등)
- 이 경우 요약 머리에 **production 버전 경로**를 명시하고 "라인 주석은 production 버전 참조"로 안내

### 요약만 분류 시 추가 동작

- 파일별 요약 머리에 **인용 박스로 라인 주석 생략 이유 명시**:
  ```markdown
  > **이 파일은 라인 주석을 만들지 않습니다.** <이유>
  ```
- 요약 본문에 "**같은 폴더의 다른 참조 파일**" 절을 두고 자매 자동/백업 파일 안내.

## 4. 건너뜀 — 바이너리·아카이브

### 확장자

| 카테고리 | 확장자 |
|---|---|
| 이미지 | `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`, `.bmp`, `.tiff`, `.ico` |
| 폰트 | `.ttf`, `.woff`, `.woff2`, `.otf`, `.eot` |
| 문서 | `.pdf`, `.pptx`, `.docx`, `.xlsx`, `.hwp`, `.hwpx` |
| 아카이브 | `.zip`, `.tar`, `.gz`, `.rar`, `.7z` |
| 미디어 | `.mp4`, `.mp3`, `.wav`, `.mov`, `.avi` |
| 데이터 | `.db`, `.sqlite`, `.parquet`, `.h5`, `.hdf5` |
| 컴파일 | `.pyc`, `.class`, `.o`, `.exe`, `.dll`, `.so` |

### 건너뜀 분류 시 동작

- 산출물 생성 안 함.
- 최종 보고서의 "건너뛴 파일" 섹션에 경로와 이유 기록.

## 5. 분류 결과 보고 양식

작업 시작 직전 사용자에게 한 번 보여줌:

```markdown
## 파일 스캔 결과

총 N개 파일 발견.

### 정상 처리 (M개)
- HTML: a개
- CSS: b개
- JS/JSON: c개
- ...

### 요약만 (X개)
- `path/to/file` — 사유: 외부 백업본 (saved from url 마커)
- `path/to/file` — 사유: 운영 사이트 원본 모형 (production: path/to/v2/file)

### 건너뜀 (Y개)
- 이미지 i개, 폰트 f개, 문서 d개

진행합니다.
```

---

## 6. 분류 우선순위 (충돌 시)

여러 규칙에 동시 해당하면 다음 우선순위로 결정:

1. **건너뜀** (바이너리는 무조건 건너뜀, 다른 규칙 무시)
2. **요약만 — 5000줄 이상** (절대 요약만)
3. **요약만 — 헤더 마커**
4. **요약만 — 폴더 위치**
5. **요약만 — 크기 + 의존도**
6. **요약만 — 원본 모형 + production 존재**
7. **정상 처리** (위 규칙 모두 해당 안 됨)

---

## 7. 사용자 직접 지정

사용자가 SKILL 호출 시 명시적으로 지정한 처리 방식이 모든 자동 규칙보다 우선.

예:
- "uploads 폴더는 그래도 라인 주석 만들어줘" → uploads 안 파일을 정상 처리로 강제.
- "5000줄짜리 .html 요약도 생략해줘" → 건너뜀으로 강등.

이런 사용자 지정은 SKILL.md의 "주의 사항"에 따라 단 한 번 받고 자동 진행.
