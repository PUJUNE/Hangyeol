# readme-seed.md — `01_읽는법.md`의 기본 골격

이 문서는 스킬이 `01_읽는법.md`를 자동 생성할 때 사용하는 **기본 골격(seed)**임. 실제 발견된 파일 타입에 맞춰 절을 추가·축소.

---

## 사용 규칙

1. 입력 폴더 스캔 결과로 **발견된 확장자 목록**을 만든다.
2. 아래 골격에서 **발견되지 않은 언어 절은 통째로 삭제**.
3. **발견된 언어 절만** 결과 `01_읽는법.md`에 포함.
4. "자주 나오는 기호" 표는 발견된 언어의 기호만 남김.
5. 추가 언어(R&D 확장 시 Python 등)가 발견되면 새 절 추가.

## 2. 파일 종류별 역할 (가장 중요)

| 확장자 | 비유 | 하는 일 |
|---|---|---|
| `.html` | **건물의 뼈대(철근)** | 화면에 무엇을 보여줄지 |
| `.css` | **건물의 페인트와 가구 배치** | 어떻게 꾸밀지 |
| `.js` | **건물의 전기 배선과 자동문** | 사용자가 클릭·스크롤하면 무엇이 움직이는지 |
| `.json` | **창고의 재고 장부** | 표·목록 형태의 데이터 |
| `.xml` | 같은 장부지만 옛날 양식 | 검색엔진이 읽기 좋은 데이터 |
| `.txt` | 그냥 종이 메모 | 사람·기계 모두 읽는 평문 |
| `.md` (Markdown) | **워드 문서의 초간단판** | 제목·목록·강조를 기호 몇 개로 표현 |
| `.py` | (R&D 확장 시) **실험 프로토콜** | 데이터 분석·자동화 명령 묶음 |
| `.ipynb` | **실험 노트** | 셀 단위 코드+설명+그래프 혼합 |
| `.r` | **통계 처리 명령서** | 통계·데이터 분석 |
| `.m` | **MATLAB 시뮬레이션** | 수치 계산·신호 처리 |

각 파일의 역할은 분리되어 있고, 서로를 호출함. 내용을 바꾸려면 데이터 파일, 색·여백을 바꾸려면 스타일 파일, 동작을 바꾸려면 스크립트 파일.

## 3. HTML 빨리 읽는 법

(발견 시에만 포함)

HTML은 **꺾쇠 괄호 `<...>`로 감싼 태그**들이 모인 문서임.

```html
<h1>회사 소개</h1>
<p>저희는 의료기기를 만듭니다.</p>
```

- `<h1>...</h1>` → 큰 제목 (h는 heading)
- `<p>...</p>` → 한 단락 (p는 paragraph)
- 여는 태그 `<h1>`과 닫는 태그 `</h1>`가 한 쌍.

**자주 나오는 태그:**

| 태그 | 비유 | 뜻 |
|---|---|---|
| `<html>` | 책 한 권 전체 | 문서 전체 바깥 |
| `<head>` | 책의 판권면 | 페이지 정보(제목·언어·아이콘) |
| `<body>` | 책의 본문 | 화면에 보이는 내용 전부 |
| `<header>` | 페이지 머리띠 | 위쪽 (로고·메뉴) |
| `<footer>` | 페이지 아래 띠 | 아래쪽 (회사 정보·저작권) |
| `<nav>` | 안내 표지판 | 메뉴 모음 |
| `<main>` | 본 매장 | 페이지의 주된 내용 영역 |
| `<section>` `<div>` | 박스 칸막이 | 묶음 그릇 |
| `<a href="...">` | 표지판 화살표 | 다른 페이지 링크 |
| `<img src="..." alt="...">` | 액자 | 그림 |
| `<button>` | 누름버튼 | 클릭 가능 |
| `<form>` `<input>` | 신청서 | 사용자 입력 양식 |
| `<ul>` `<li>` | 글머리표 목록 | 점·번호 나열 |
| `<svg>` | 도형 그림 | 작은 아이콘 |
| `<meta>` | 도서관 카드 | 검색엔진용 정보 |
| `<link>` | 부속 부품 호출장 | CSS 등 다른 파일 |
| `<script>` | 자동 장치 호출장 | JS 파일 |

**속성(attribute):**

```html
<a href="/about.html" class="active" id="main-link">회사소개</a>
```

- `href="..."` → 갈 곳
- `class="..."` → CSS·JS가 잡는 꼬리표 (여러 요소 가능)
- `id="..."` → 한 페이지에 단 하나뿐인 이름표

`class`와 `id`는 **CSS와 JS가 "어느 부분을 꾸밀까/움직일까"를 찾는 손잡이**.

## 4. CSS 빨리 읽는 법

(발견 시에만 포함)

```css
.bp-header {
  background: #ffffff;
  padding: 16px 24px;
  border-bottom: 1px solid #d9d9d9;
}
```

- `.bp-header` → 꼬리표가 `bp-header`인 요소
- `{ ... }` → 이렇게 꾸며
- `background: #ffffff;` → 배경 흰색
- `padding: 16px 24px;` → 안쪽 여백 위아래 16, 좌우 24

**선택자 기호:**

| 기호 | 의미 |
|---|---|
| `.이름` | class가 `이름`인 요소 |
| `#이름` | id가 `이름`인 요소 |
| `태그명` | 그 태그 전부 |
| `A B` (공백) | A 안에 든 B |
| `A, B` | A 또는 B |
| `:hover` | 마우스 올렸을 때 |
| `@media` | 화면 크기에 따라 |

색상은 `#ffffff` 코드, 크기는 `px`·`rem`·`%`.

## 5. JavaScript 빨리 읽는 법

(발견 시에만 포함)

```js
button.addEventListener('click', () => {
  alert('안녕하세요');
});
```

→ "그 button을 클릭하면, '안녕하세요' 알림창을 띄워라"

| 모양 | 뜻 |
|---|---|
| `const 이름 = 값;` | 이 값을 이 이름으로 (바뀌지 않음) |
| `let 이름 = 값;` | 같은 뜻이지만 바뀔 수 있음 |
| `function 이름(인자) { ... }` | 작업 묶음 정의 |
| `이름(인자)` | 작업 묶음 실행 |
| `() => { ... }` | 이름 없는 작업 묶음 (화살표 함수) |
| `if (조건) { ... } else { ... }` | 만약 ~이면 ~ |
| `요소.addEventListener('이벤트', 함수)` | "이 요소에서 ~ 일이 일어나면 ~" |
| `document.getElementById('이름')` | id가 "이름"인 HTML 요소 |
| `document.querySelectorAll('.이름')` | class가 "이름"인 모든 요소 |
| `fetch(주소).then(...)` | 인터넷에서 데이터 받아 처리 |
| `localStorage.getItem('키')` | 사용자 컴퓨터에 저장된 값 |
| `// 글자` | 사람용 메모 |

문장 끝의 **세미콜론(`;`)**은 마침표. 중괄호 `{ ... }`는 한 묶음 행동, 소괄호 `( ... )`는 입력값.

## 6. JSON 빨리 읽는 법

(발견 시에만 포함)

```json
{
  "name": "심전도 전극",
  "price": 12000,
  "tags": ["ECG", "전극"],
  "available": true
}
```

- `{ ... }` → 묶음(객체)
- `"이름": 값` → 이름과 값의 짝
- 값은 글자(`"..."`), 숫자(`12000`), 참/거짓(`true`/`false`), 목록(`[...]`)
- `[ ... , ... ]` → 목록(배열)
- 모든 이름은 큰따옴표
- 마지막 항목 뒤에 콤마 금지

## 8. 자주 나오는 기호 한 줄 사전

(발견된 언어의 기호만 표에 남김)

| 기호 | 어디서 | 뜻 |
|---|---|---|
| `<...>` | HTML | 태그 |
| `</...>` | HTML | 닫는 태그 |
| `<!-- ... -->` | HTML | 주석 |
| `{ ... }` | CSS·JS·JSON | 묶음 |
| `[ ... ]` | JS·JSON | 목록(배열) |
| `( ... )` | JS·CSS | 입력값 |
| `;` | CSS·JS | 마침표 |
| `:` | CSS·JS·JSON | 이름과 값 잇기 |
| `,` | 어디서나 | 항목 구분 |
| `"..."` `'...'` | 어디서나 | 글자(문자열) |
| `//` | JS | 사람용 메모 |
| `/* ... */` | JS·CSS | 여러 줄 사람용 메모 |
| `#` | CSS | id / 색 코드 |
| `.` | CSS·JS | class / "이 안의 ~" |
| `=` | HTML 속성 / JS 대입 | 같다·정한다 |
| `===` | JS | 정확히 같다 |
| `&&` `\|\|` | JS | 그리고·또는 |
| `=>` | JS | 화살표 함수 |

## 9. 이 폴더의 산출물 읽는 순서

1. **이 문서 (`01_읽는법.md`)** — 한 번 읽고 옆에 둠
2. **`00_아키텍처.md`** — 사이트 전체 지도
3. **`02_파일별_요약/<원본경로>.md`** — 한 파일을 통째로 이해할 때
4. **`03_라인주석/<원본경로>`** — 한 줄 한 줄이 무엇을 하는지 봐야 할 때 (디버깅·수정 시점)

### 파일별 요약 읽는 법

각 요약은 두 부분으로 구성:
- **① 전체 요약** — 이 파일이 사이트 안에서 어떤 역할을 맡는가, 비유 사용.
- **② 의미 단위 묶음** — 2~5줄 묶음으로 쪼개어 "이 묶음은 무엇을 하는가"를 비유와 함께. 각 묶음마다 원본 라인 번호 표기.

### 라인 주석본 읽는 법

원본과 같은 모양 그대로, 모든 줄 끝(또는 위)에 한국어 한 줄 주석. 주석에는 **그 줄이 무엇을 하는지만** 적혀 있음 — 비유나 배경 설명은 파일별 요약에서.

## 10. 처음 읽는 사람을 위한 권장 순서

1. 이 문서 (`01_읽는법.md`) 한 번 통독
2. `00_아키텍처.md` 한 번 통독
3. `02_파일별_요약/<가장 큰 페이지>.md` 한 번 통독
4. 궁금한 줄이 있으면 `03_라인주석/`의 같은 줄 펴서 확인
5. 같은 방식으로 다른 파일로 확장

이 순서대로 읽으면 **한 줄도 빼놓지 않고 사이트가 굴러가는 원리**가 머리에 잡힘. 그 상태가 되면 AI에게 "이 부분을 이렇게 바꿔줘"라고 정확히 의지를 표현할 수 있게 됨.
