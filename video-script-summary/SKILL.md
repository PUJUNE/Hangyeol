# 영상 스크립트 → 마크다운 정리보고서

## 입력 형식 3종

### (A) 스크립트 파일 — `.txt` / `.srt`
* 타임코드(`0:00`, `00:00:00,000` 등) 포함 가능
* UTF-8 인코딩 권장 (CP949 등은 자동 변환 시도)

### (B) 유튜브 URL — `https://www.youtube.com/watch?v=...` 또는 `https://youtu.be/...`
* `scripts/fetch_youtube_subs.py`로 자막·메타데이터 자동 추출
* 자막 우선순위: 한국어 → 영어 → 자동 생성된 자막
* 메타데이터: 영상 제목, 채널명, 업로드일, 러닝타임, URL → 보고서 1번 섹션 자동 채움
* 자막이 전혀 없는 영상: 사용자에게 알리고 작업 중단 (Whisper 등 STT는 본 스킬 범위 밖)

### (C) 기존 `.md` 보고서 보강
* 이미 작성된 보고서를 9섹션 양식에 맞춰 누락 섹션 채우기

## 출력
* `<원본파일명_또는_영상제목>_정리보고서.md`

## 보고서 구조 (필수 9개 섹션)

```markdown
# <영상 제목> 정리보고서

## 1. 영상 개요
- 출처(URL)
- 채널명 · 업로드일 (URL 입력 시)
- 주제 (1~2문장)
- 러닝타임
- 발화자 결론 요약 (1문장)

## 2. 도입부 — 현상 제시
### 2.1 사건의 배경
### 2.2 발화자가 지목한 핵심 관찰

## 3. 기존 통념에 대한 반박
### 3.1 일반적 해석 (발화자가 비판하는 관점)
### 3.2 발화자의 반박 논리

## 4. 핵심 논거
- 결과가 아닌 과정/구조 중심으로 분해
- 비교가 가능한 항목은 마크다운 표로 정리

## 5. 이론적 뒷받침
- 발화자가 인용한 학자·이론·역사적 사례 포함
- 출처가 불명확하면 "출처 미확인"으로 표기함

## 6. 한국 특수성 / 맥락 (해당 시)
- 영상이 특정 사회·문화 맥락을 다룰 때만 작성

## 7. 종합 결론
- 발화자의 최종 메시지 3~5개 bullet

## 8. 영상의 논리 구조 요약
- 텍스트 ASCII 다이어그램으로 흐름 도식화
  예시:
  ```
  [현상 제시] → [기존 해석 비판] → [대안 해석]
        ↓
  [이론적 근거] → [최종 결론]
  ```

## 9. 핵심 인용구
- 영상에서 직접 인용 가능한 문장 3~5개를 그대로 발췌 (>) 인용 블록으로
```

## 문체
* ~음 체로 통일함 (예: ~확인됨, ~있음, ~판단됨)

## 작성 원칙
* 화자가 명시적으로 말한 내용만 정리할 것. 추론·확장 금지
* 비교 가능한 항목은 표로 시각화할 것 (현실 vs 게임, 이전 vs 이후 등)
* 인용구는 원문을 압축하되 의미 변형 금지. 변형 시 "[요약]" 표기
* 출처가 모호한 학술적 주장은 "출처 미확인"으로 표기
* 타임코드는 보고서 본문에서 제거하되, 인용구에 한해 `(타임코드)`를 괄호로 부기 가능

## 처리 절차

### 공통
1. 입력 형식 확인 (A/B/C)
2. 형식별 전처리(자막 다운로드 또는 인코딩 검증)
3. 9섹션 양식에 따라 보고서 작성
4. `<파일명>_정리보고서.md`로 저장

### (A) 스크립트 파일 처리
1. 파일 인코딩(UTF-8) 정상 여부 확인
2. 타임코드·발화 노이즈("어", "그", 단순 추임새) 제거
3. 발화 전체를 5단계 논증 구조로 매핑(도입·현상·통념·반박·결론)
4. 각 섹션을 ~음 체 bullet로 변환
5. 핵심 인용구 추출 후 마지막 섹션에 배치

### (B) 유튜브 URL 처리
1. `python scripts/fetch_youtube_subs.py <URL> --out <작업폴더>` 실행
2. 출력: `<영상제목>_스크립트.txt` (자막 본문) + `<영상제목>_meta.json` (메타데이터)
3. 자막 본문을 (A) 절차로 처리하되, 1번 섹션의 출처·채널명·업로드일·러닝타임은 meta.json에서 자동 채움
4. 한글 자막이 없고 영어 자막만 있을 경우: 사용자에게 한글 번역 후 진행 여부 확인
5. 자막이 전혀 없는 영상: 작업 중단 + 사용자에게 통지

### (C) 기존 보고서 보강
1. 기존 .md 파일을 Read로 로드하여 9섹션 양식 누락 항목 식별
2. 누락 항목만 채움. 기존 내용은 보존
3. 중복 섹션은 통합

## 의존성 (B 형식만)
* `yt-dlp` (필수): `pip install yt-dlp` (또는 `pip install yt-dlp --break-system-packages`)
* `ffmpeg` (선택, 자막 형식 변환에만 필요): 시스템 설치
* 미설치 시 fetch_youtube_subs.py가 명확한 에러 메시지 출력 후 사용자에게 설치 안내

## 분량 제한
* 본문 기준 1,500~2,500단어 권장 (원본 분량의 1/4~1/3)
* 너무 짧으면 추론 위험, 너무 길면 단순 받아쓰기 위험

## 모호한 입력 처리
* 영상 출처(URL) 없을 때 (A 형식): 사용자에게 확인 요청
* 발화자가 누구인지 불명확할 때: "발화자"로 통일 표기
* 스크립트가 다중 화자일 경우: 사용자에게 화자 식별 정보 요청
* 유튜브 URL이 비공개·삭제·연령제한 영상일 때 (B 형식): yt-dlp 에러 그대로 사용자에게 보고 후 중단

## 사용 예
```bash
# 형식 B (유튜브 URL)
python scripts/fetch_youtube_subs.py "https://www.youtube.com/watch?v=XXXXX" --out ./
# → <영상제목>_스크립트.txt + <영상제목>_meta.json 생성
# 이후 일반 보고서 작성 절차 진입
```
